| Step                          | Description                                                               |
| ----------------------------- | ------------------------------------------------------------------------- |
| 💡 **Start with an Idea**     | Identify a problem, opportunity, or improvement                           |
| 🧩 **Feature Card**           | Define the unit of work (clear, scoped, outcome-driven)                   |
| 📄 **PRD**                    | Expand into structured requirements and success criteria                  |
| 🛠️ **Build Prompt**          | Translate PRD into executable instructions for the builder (e.g., Replit) |
| ⚙️ **Execute**                | Build and deploy                                                          |
| ✅ **Review Against Done**     | Validate against acceptance criteria (not vibes)                          |
| 📚 **Capture Learning**       | Document what worked, what didn’t, gaps                                   |
| 🔄 **Feed Next Step Back In** | Use learning to drive the next iteration                                  |
